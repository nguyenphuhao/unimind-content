Key Component Props
ContentShell Wrapper chính, 1 lần duy nhất trong layout.tsx
PropTypeDescription
childrenReactNodePage content
Auto-detect embedded mode từ searchParams hoặc headers. Nếu embedded → ẩn SiteHeader + SiteFooter, chỉ render children.
ArticleLayout Blog detail — centered prose column
PropTypeDescription
headerReactNodeArticleHeader slot
childrenReactNodeMDX content
footerReactNodeArticleFooter slot
max-width: 720px, centered, leading-relaxed cho prose. Padding responsive: 16px mobile → 24px tablet → 0 desktop.
DocsLayout Wiki/Handbook detail — content + sidebar TOC
PropTypeDescription
headerReactNodeArticleHeader slot
childrenReactNodeMDX content
footerReactNodeArticleFooter slot
headingsHeading[]Extracted headings cho TOC
Desktop (lg+): content left + sticky TOC right (240px). Mobile: inline TOC above content. max-width: 1024px.
ListFeed Listing page — reusable cho cả 3 content types
PropTypeDescription
itemsFeedItem[]Array of content items
groupBystring?Optional: group by field (category, section)
showOrderboolean?Show numbered order (handbook)
basePathstringLink prefix: /en/blog, /vi/wiki...
Blog: flat list, show tags + read time. Wiki: groupBy="category". Handbook: groupBy="section" + showOrder=true.
TableOfContents Auto-generated từ MDX headings
PropTypeDescription
headingsHeading[]{ id, text, level }
variant"sidebar" | "inline"Render mode based on viewport
Sidebar: sticky top-[80px], scroll-spy highlight active heading. Inline: collapsible, show above content on mobile.
